# 检索功能方案 - 通用模块架构设计

## 一、设计目标

### 1.1 核心原则
- ✅ **通用性**：不绑定具体业务实体（Note、Tag等），可适用于任何项目
- ✅ **独立性**：作为独立的 Android Library 模块，可单独使用
- ✅ **可配置性**：分词器、索引字段、词典管理都可配置
- ✅ **可扩展性**：支持自定义分词器、自定义字段类型
- ✅ **易用性**：提供简洁的 API，最小化集成成本

## 二、核心接口设计

### 2.1 Searchable 接口（可搜索对象）

**设计理念**：任何需要被搜索的对象都实现此接口

```java
package com.yourcompany.search.core;

/**
 * 可搜索对象接口
 * 业务实体需要实现此接口才能被索引和搜索
 */
public interface Searchable {
    /**
     * 获取唯一标识符
     */
    String getId();
    
    /**
     * 获取可搜索的文本内容
     * @return 文本内容列表，每个元素对应一个搜索字段
     */
    Map<String, String> getSearchableFields();
    
    /**
     * 获取数值字段（用于排序和过滤）
     * @return 字段名 -> 数值的映射
     */
    Map<String, Long> getNumericFields();
    
    /**
     * 获取标签/关键词列表（用于词典管理）
     * @return 标签列表，这些标签会被自动加入词典
     */
    List<String> getTags();
}
```

**使用示例**：
```java
// Note 实体实现 Searchable
public class Note implements Searchable {
    private long id;
    private String content;
    private List<String> tags;
    private long timestamp;
    
    @Override
    public String getId() {
        return String.valueOf(id);
    }
    
    @Override
    public Map<String, String> getSearchableFields() {
        Map<String, String> fields = new HashMap<>();
        fields.put("content", content);
        // 可以添加更多字段
        return fields;
    }
    
    @Override
    public Map<String, Long> getNumericFields() {
        Map<String, Long> fields = new HashMap<>();
        fields.put("timestamp", timestamp);
        return fields;
    }
    
    @Override
    public List<String> getTags() {
        return tags != null ? tags : Collections.emptyList();
    }
}
```

### 2.2 SearchEngine 接口（搜索引擎主接口）

```java
package com.yourcompany.search.core;

/**
 * 搜索引擎接口
 * 提供索引和搜索功能
 */
public interface SearchEngine {
    /**
     * 初始化搜索引擎
     * @param config 索引配置
     */
    void initialize(IndexConfig config);
    
    /**
     * 添加或更新文档到索引
     * @param searchable 可搜索对象
     */
    void index(Searchable searchable);
    
    /**
     * 批量添加或更新文档
     * @param searchables 可搜索对象列表
     */
    void indexBatch(List<? extends Searchable> searchables);
    
    /**
     * 从索引中删除文档
     * @param id 文档ID
     */
    void delete(String id);
    
    /**
     * 搜索
     * @param query 搜索关键词
     * @param maxResults 最大结果数
     * @return 搜索结果列表
     */
    List<SearchResult> search(String query, int maxResults);
    
    /**
     * 高级搜索
     * @param options 搜索选项
     * @return 搜索结果列表
     */
    List<SearchResult> search(SearchOptions options);
    
    /**
     * 重建索引（从数据源重新构建）
     * @param dataProvider 数据提供者
     */
    void rebuildIndex(SearchableDataProvider dataProvider);
    
    /**
     * 关闭搜索引擎
     */
    void close();
}
```

## 三、实现层设计

### 3.1 LuceneSearchEngine（Lucene 实现）

```java
package com.yourcompany.search.lucene;

public class LuceneSearchEngine implements SearchEngine {
    private IndexConfig config;
    private IndexWriter writer;
    private IndexSearcher searcher;
    private Directory directory;
    
    @Override
    public void initialize(IndexConfig config) {
        this.config = config;
        // 初始化 Lucene 索引
        directory = FSDirectory.open(config.getIndexDirectory());
        IndexWriterConfig writerConfig = new IndexWriterConfig(config.getAnalyzer());
        writer = new IndexWriter(directory, writerConfig);
        // ...
    }
    
    @Override
    public void index(Searchable searchable) {
        Document doc = buildDocument(searchable);
        writer.updateDocument(new Term("id", searchable.getId()), doc);
    }
    
    private Document buildDocument(Searchable searchable) {
        Document doc = new Document();
        doc.add(new StringField("id", searchable.getId(), Field.Store.YES));
        
        // 添加文本字段
        for (Map.Entry<String, String> entry : searchable.getSearchableFields().entrySet()) {
            FieldConfig fieldConfig = config.getFieldConfig(entry.getKey());
            if (fieldConfig != null && fieldConfig.getType() == FieldType.TEXT) {
                TextField field = new TextField(entry.getKey(), entry.getValue(), 
                    fieldConfig.isStored() ? Field.Store.YES : Field.Store.NO);
                field.setBoost(fieldConfig.getBoost());
                doc.add(field);
            }
        }
        
        // 添加数值字段
        for (Map.Entry<String, Long> entry : searchable.getNumericFields().entrySet()) {
            doc.add(new LongPoint(entry.getKey(), entry.getValue()));
            doc.add(new StoredField(entry.getKey(), entry.getValue()));
        }
        
        return doc;
    }
    
    // ... 其他方法实现
}
```

## 四、业务层适配（NotePlus 项目）

### 4.1 NoteSearchable（Note 实体适配）

```java
package person.notfresh.noteplus.search;

import com.yourcompany.search.core.Searchable;
import person.notfresh.noteplus.db.NoteDbHelper;

public class NoteSearchable implements Searchable {
    private long noteId;
    private String content;
    private List<String> tags;
    private long timestamp;
    private String commentContent; // 评论内容（可选）
    
    // 从数据库 Cursor 构建
    public static NoteSearchable fromCursor(Cursor cursor, NoteDbHelper dbHelper) {
        NoteSearchable note = new NoteSearchable();
        note.noteId = cursor.getLong(cursor.getColumnIndex(NoteDbHelper.COLUMN_ID));
        note.content = cursor.getString(cursor.getColumnIndex(NoteDbHelper.COLUMN_CONTENT));
        note.timestamp = cursor.getLong(cursor.getColumnIndex(NoteDbHelper.COLUMN_TIMESTAMP));
        
        // 加载标签
        Cursor tagCursor = dbHelper.getTagsForNote(note.noteId);
        note.tags = new ArrayList<>();
        while (tagCursor.moveToNext()) {
            note.tags.add(tagCursor.getString(tagCursor.getColumnIndex(NoteDbHelper.COLUMN_TAG_NAME)));
        }
        tagCursor.close();
        
        // 加载评论内容（可选）
        // ...
        
        return note;
    }
    
    @Override
    public String getId() {
        return String.valueOf(noteId);
    }
    
    @Override
    public Map<String, String> getSearchableFields() {
        Map<String, String> fields = new HashMap<>();
        fields.put("content", content);
        if (commentContent != null && !commentContent.isEmpty()) {
            fields.put("comment", commentContent);
        }
        // 标签也作为可搜索字段
        if (tags != null && !tags.isEmpty()) {
            fields.put("tags", String.join(" ", tags));
        }
        return fields;
    }
    
    @Override
    public Map<String, Long> getNumericFields() {
        Map<String, Long> fields = new HashMap<>();
        fields.put("timestamp", timestamp);
        return fields;
    }
    
    @Override
    public List<String> getTags() {
        return tags != null ? tags : Collections.emptyList();
    }
}
```

### 4.2 TagDictionaryProvider（标签词典提供者）

```java
package person.notfresh.noteplus.search;

import com.yourcompany.search.dictionary.DictionaryProvider;
import person.notfresh.noteplus.db.NoteDbHelper;

public class TagDictionaryProvider implements DictionaryProvider {
    private NoteDbHelper dbHelper;
    private DictionaryChangeListener listener;
    
    public TagDictionaryProvider(NoteDbHelper dbHelper) {
        this.dbHelper = dbHelper;
    }
    
    @Override
    public List<String> getDictionaryWords() {
        List<String> tags = new ArrayList<>();
        Cursor cursor = dbHelper.getAllTags();
        while (cursor.moveToNext()) {
            tags.add(cursor.getString(cursor.getColumnIndex(NoteDbHelper.COLUMN_TAG_NAME)));
        }
        cursor.close();
        return tags;
    }
    
    @Override
    public void setChangeListener(DictionaryChangeListener listener) {
        this.listener = listener;
    }
    
    // 当标签变化时，通知监听器
    public void notifyTagAdded(String tag) {
        if (listener != null) {
            listener.onWordsAdded(Collections.singletonList(tag));
        }
    }
    
    public void notifyTagRemoved(String tag) {
        if (listener != null) {
            listener.onWordsRemoved(Collections.singletonList(tag));
        }
    }
}
```

### 4.3 NoteSearchEngine（业务层封装）

```java
package person.notfresh.noteplus.search;

import com.yourcompany.search.core.*;
import com.yourcompany.search.lucene.LuceneSearchEngine;
import person.notfresh.noteplus.db.NoteDbHelper;

public class NoteSearchEngine {
    private SearchEngine searchEngine;
    private NoteDbHelper dbHelper;
    private DictionaryManager dictManager;
    
    public void initialize(Context context, NoteDbHelper dbHelper) {
        this.dbHelper = dbHelper;
        
        // 创建词典管理器
        TagDictionaryProvider tagProvider = new TagDictionaryProvider(dbHelper);
        dictManager = new FileDictionaryManager();
        dictManager.initialize(context, tagProvider);
        
        // 创建索引配置
        IndexConfig config = new IndexConfig.Builder()
            .setIndexDirectory(new File(context.getFilesDir(), "lucene_index"))
            .setAnalyzer(dictManager.createAnalyzer())
            .setDictionaryManager(dictManager)
            .addField(new FieldConfig("content", FieldType.TEXT, true, true, 1.0f))
            .addField(new FieldConfig("tags", FieldType.TEXT, false, true, 1.5f)) // 标签权重更高
            .addField(new FieldConfig("comment", FieldType.TEXT, false, true, 0.8f))
            .addField(new FieldConfig("timestamp", FieldType.NUMERIC, true, true, 1.0f))
            .build();
        
        // 创建搜索引擎
        searchEngine = new LuceneSearchEngine();
        searchEngine.initialize(config);
    }
    
    // 业务层便捷方法
    public void indexNote(long noteId) {
        // 从数据库加载 Note，转换为 Searchable，然后索引
        Cursor cursor = dbHelper.getNoteById(noteId);
        if (cursor.moveToFirst()) {
            NoteSearchable searchable = NoteSearchable.fromCursor(cursor, dbHelper);
            searchEngine.index(searchable);
        }
        cursor.close();
    }
    
    public List<SearchResult> searchNotes(String query) {
        return searchEngine.search(query, 50);
    }
    
    // ... 其他业务方法
}
```

## 六、优势分析

### 6.1 通用性
- ✅ 不绑定具体业务实体，任何实现 `Searchable` 的对象都可以被搜索
- ✅ 适用于笔记、文档、商品、文章等任何需要全文搜索的场景

### 6.2 独立性
- ✅ 作为独立的 Android Library，可以单独编译、测试、发布
- ✅ 可以轻松集成到其他项目中

### 6.3 可配置性
- ✅ 分词器可替换（IK Analyzer、SmartChineseAnalyzer 等）
- ✅ 索引字段可配置（字段名、类型、权重等）
- ✅ 词典管理可扩展（支持多种词典数据源）

### 6.4 可扩展性
- ✅ 可以轻松添加新的搜索引擎实现（如 SQLite FTS5）
- ✅ 可以自定义分词器
- ✅ 可以添加新的字段类型

### 6.5 易用性
- ✅ 业务层只需要实现 `Searchable` 接口
- ✅ 提供 Builder 模式，配置简单
- ✅ 业务层封装类（如 `NoteSearchEngine`）提供便捷方法

## 九、总结

这个通用模块架构设计具有以下特点：

1. **高度解耦**：核心模块不依赖业务逻辑，业务层通过接口适配
2. **易于复用**：其他项目只需实现 `Searchable` 接口即可使用
3. **灵活配置**：支持自定义分词器、字段配置、词典管理等
4. **易于扩展**：可以轻松添加新的搜索引擎实现或功能
5. **清晰的层次**：核心层、实现层、业务层职责分明

这样的设计既满足了当前项目的需求（标签词典、分词查看等），又保证了模块的通用性和可复用性。
