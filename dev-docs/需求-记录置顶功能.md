# 需求：记录置顶功能

## 功能描述
允许用户将重要的笔记记录置顶显示，置顶的记录会始终显示在列表的最前面，方便快速查看。

## 功能需求

### 1. 置顶状态管理
- 每条笔记可以设置为置顶或取消置顶
- 置顶状态需要持久化保存到数据库
- 置顶状态在切换项目时保持独立（每个项目有自己的置顶记录）

### 2. 显示规则
- 置顶的记录始终显示在列表最前面
- 置顶记录之间按时间排序（遵循项目的排序设置：正序或逆序）
- 非置顶记录按时间排序显示在置顶记录之后
- 置顶的记录在内容前显示"📌 "标识

### 3. 操作方式
- 在笔记的长按菜单中添加"置顶"/"取消置顶"选项
- 点击后切换置顶状态，并刷新列表
- 显示操作提示（"已置顶"或"已取消置顶"）

## 技术实现方案

### 第一阶段：数据库设计

#### 1.1 数据库字段
- 在 `notes` 表中添加 `is_pinned` 字段（INTEGER类型，0表示未置顶，1表示置顶）
- 默认值为 0（未置顶）

#### 1.2 数据库升级
- 更新 `DATABASE_VERSION` 从 6 到 7
- 在 `onUpgrade` 方法中添加升级逻辑：
  ```sql
  ALTER TABLE notes ADD COLUMN is_pinned INTEGER DEFAULT 0
  ```

#### 1.3 常量定义
在 `NoteDbHelper` 中添加：
```java
public static final String COLUMN_IS_PINNED = "is_pinned";
```

### 第二阶段：数据库操作方法

#### 2.1 切换置顶状态方法
在 `NoteDbHelper` 中添加：
```java
/**
 * 切换笔记的置顶状态
 * @param noteId 笔记ID
 * @return 是否成功
 */
public boolean togglePinNote(long noteId)
```

实现逻辑：
1. 查询当前笔记的 `is_pinned` 值
2. 如果为 0，更新为 1；如果为 1，更新为 0
3. 返回更新是否成功

#### 2.2 查询置顶状态方法
在 `NoteDbHelper` 中添加：
```java
/**
 * 获取笔记的置顶状态
 * @param noteId 笔记ID
 * @return true表示置顶，false表示未置顶
 */
public boolean isNotePinned(long noteId)
```

### 第三阶段：UI显示逻辑

#### 3.1 修改排序逻辑
在 `MainActivity.loadMoments()` 方法中：
- 修改 `orderBy` 语句，先按 `is_pinned DESC` 排序，再按时间排序
- 查询时包含 `COLUMN_IS_PINNED` 字段

排序规则：
```sql
ORDER BY is_pinned DESC, timestamp DESC/ASC
```
- `is_pinned DESC`：置顶记录（值为1）排在前面
- `timestamp DESC/ASC`：根据项目设置的时间排序方式

#### 3.2 显示置顶标识
在 `MainActivity.getView()` 方法中：
- 读取 `is_pinned` 字段值
- 如果为 1，在内容文本前添加"📌 "标识
- 注意：需要处理折叠功能、花费显示等场景，确保标识正确显示

显示位置：
- 在 `updateListItemWithExtras()` 之后设置置顶标识
- 在 `checkAndShowFoldButton()` 中也需要考虑置顶标识

#### 3.3 折叠功能中的置顶标识处理
在 `checkAndShowFoldButton()` 方法中：
- 检查当前文本是否已有置顶标识
- 如果有，在设置折叠/展开文本时保留标识
- 确保标识始终在最前面

### 第四阶段：操作菜单

#### 4.1 添加菜单选项
在 `showNoteOptionsMenu()` 方法中：
- 查询当前笔记的置顶状态
- 根据状态显示"置顶"或"取消置顶"
- 菜单项顺序：["复制到剪切板", "追加内容", "置顶/取消置顶", "删除"]

#### 4.2 切换置顶状态
在 `MainActivity` 中添加：
```java
/**
 * 切换笔记的置顶状态
 * @param noteId 笔记ID
 */
private void togglePinNote(long noteId)
```

实现逻辑：
1. 调用 `dbHelper.togglePinNote(noteId)` 切换状态
2. 显示操作提示（"已置顶"或"已取消置顶"）
3. 调用 `loadMoments()` 刷新列表

### 第五阶段：边界情况处理

#### 5.1 新建笔记
- 新建笔记时，`is_pinned` 字段使用默认值 0（未置顶）
- 不需要特殊处理

#### 5.2 删除笔记
- 删除笔记时，置顶状态会随记录一起删除
- 不需要特殊处理

#### 5.3 移动笔记到其他项目
- 移动笔记时，置顶状态会一起移动
- 在目标项目中，笔记保持原来的置顶状态

#### 5.4 导入导出
- 导出时，可以包含 `is_pinned` 字段（可选）
- 导入时，如果包含 `is_pinned` 字段，则恢复置顶状态

3. ⏳ **排序逻辑**：修改 `loadMoments()` 的排序规则
4. ⏳ **显示标识**：在列表项中显示"📌 "标识
5. ⏳ **菜单选项**：在长按菜单中添加置顶选项
6. ⏳ **切换功能**：实现切换置顶状态的方法
7. ⏳ **测试验证**：测试各种场景下的置顶功能

## 注意事项

1. **置顶标识显示**：
   - 需要确保在各种场景下（折叠、展开、花费显示）都能正确显示
   - 标识应该始终在最前面

2. **排序逻辑**：
   - 置顶记录之间也要遵循时间排序规则
   - 非置顶记录之间也要遵循时间排序规则

3. **性能考虑**：
   - 置顶字段已添加索引（通过主键）
   - 排序查询性能影响较小

4. **用户体验**：
   - 切换置顶状态后立即刷新列表
   - 显示明确的操作反馈

5. **兼容性**：
   - 旧数据升级时，`is_pinned` 字段默认为 0（未置顶）
   - 不影响现有功能
