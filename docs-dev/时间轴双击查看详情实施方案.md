# 时间轴双击查看详情实施方案

## 一、需求概述

为时间轴功能添加双击查看详情功能：
1. **双击时间轴中的内容项**，可以查看对应的Note详情（包含该Note的所有Comment）
2. **如果是Comment项**，双击后查看关联的Note详情
3. **如果是Note项**，双击后查看该Note的详情（包含所有Comment）

## 二、现有实现分析

### 2.1 时间轴数据结构

#### Comment对象结构
- `id`: Comment的ID
- `noteId`: 关联的Note ID
- `itemType`: 类型（NOTE、COMMENT、DATE_DIVIDER）
- `projectName`: 所属项目名称
- `content`: 内容
- `timestamp`: 时间戳
- `cost`: 花费金额
- `isPinned`: 是否置顶（仅对NOTE类型有效）

#### 关键方法
- `Comment.getItemType()`: 获取项类型
- `Comment.getNoteId()`: 获取关联的Note ID
- `Comment.getProjectName()`: 获取项目名称
- `Comment.isNote()`: 判断是否为Note转换来的
- `Comment.isPinned()`: 获取置顶状态

### 2.2 现有功能

#### 项目切换
- `ProjectContextManager.switchToProject(String projectName)`: 切换项目
- `MainActivity.switchProject(String projectName)`: 切换项目并刷新UI
- `switchProject()` 方法已有完整的实现，包括进度提示和错误处理

#### 数据加载
- `NoteDbHelper.getCommentsForNote(long noteId)`: 获取Note的所有Comment（返回Cursor）
- `NoteDbHelper.loadNotes()`: 加载Note列表
- `NoteDataLoader.cursorToNoteList()`: 将Cursor转换为Note列表
- `NoteListManager.loadCommentsForNote()`: 为Note加载Comment数据

#### UI交互
- 当前时间轴列表使用 `ListView.setOnItemClickListener()` 处理点击
- 已有TODO注释：需要添加跳转到Note详情的逻辑
- 时间轴对话框在 `showTimelineDialog()` 方法中实现

### 2.3 需要实现的功能

1. **双击检测机制**：区分单击和双击
2. **项目切换逻辑**：如果Note属于不同项目，需要切换项目
3. **Note详情对话框**：显示Note内容和所有Comment
4. **数据加载**：从对应项目加载Note及其所有Comment

## 三、设计方案

### 3.1 双击检测方案

**方案A：使用Handler延迟检测（推荐，选择这个，听你的建议）**
- 优点：实现简单，兼容性好，无需额外依赖
- 缺点：需要处理单击延迟
- 实现方式：通过Handler延迟执行单击操作，在延迟期间如果再次点击则视为双击

**方案B：使用GestureDetector**
- 优点：更精确的双击检测
- 缺点：需要额外的依赖，实现复杂度较高

**推荐使用方案A**，通过Handler延迟执行单击操作，在延迟期间如果再次点击则视为双击。

**技术要点：**
- 使用 `Handler` 和 `Runnable` 实现延迟检测
- 定义双击时间间隔常量（建议300ms）
- 记录上次点击时间和位置
- 在延迟期间如果再次点击相同位置，取消单击操作，执行双击操作

### 3.2 Note详情对话框设计

#### 对话框布局结构
```
标题栏（Note详情 + 关闭按钮）
├─ Note内容区域（ScrollView）
│  ├─ 项目名称标签
│  ├─ Note内容文本（支持多行，完整显示）
│  ├─ 时间显示
│  ├─ 花费金额（如果有，显示格式：花费：XX元）
│  ├─ 置顶标识（如果置顶，显示"📌 "）
│  └─ 标签显示（如果有，横向排列）
├─ Comment列表区域（ListView或RecyclerView）
│  └─ 可滚动的Comment列表
│     ├─ Comment项1（时间、内容、花费）
│     ├─ Comment项2
│     └─ ...
└─ 底部操作栏
   ├─ 追加内容按钮（可选）
   └─ 关闭按钮
```

#### 数据展示规则
- **Note内容**：完整显示，支持多行，不截断
- **Comment列表**：按时间顺序显示（正序，最旧的在前），包含时间、内容、花费
- **项目信息**：显示Note所属项目名称
- **标签信息**：显示Note的标签（如果有），样式与时间轴一致
- **置顶标识**：如果Note置顶，在内容前显示"📌 "标识

#### UI设计要点
- 对话框大小：宽度90%，高度80%（与时间轴对话框一致）
- Note内容区域：使用ScrollView支持长内容滚动
- Comment列表：使用ListView，支持滚动
- 样式：与现有对话框风格保持一致

### 3.3 项目切换逻辑

#### 切换流程
1. **检查项目一致性**
   - 获取当前项目：`projectManager.getCurrentProject()`
   - 获取目标项目：`comment.getProjectName()`
   - 比较两个项目名称

2. **如果需要切换项目**
   - 调用 `switchProject(projectName)` 方法
   - 在切换完成的回调中继续加载Note详情
   - 注意：时间轴对话框是跨项目的，不需要关闭

3. **如果不需要切换**
   - 直接加载Note详情
   - 显示详情对话框

#### 错误处理策略
- **项目不存在**：检查 `projectManager.getDbHelperForProject(projectName)` 返回null
- **Note不存在**：检查数据库查询结果为空
- **切换失败**：捕获 `switchProject()` 的异常或返回值
- **数据库错误**：捕获数据库查询异常

#### UI反馈
- 切换项目时：显示"正在切换项目..."进度提示
- 加载详情时：显示"正在加载详情..."进度提示
- 错误时：显示Toast错误提示

### 3.4 数据加载策略

#### 加载Note详情流程
1. **获取数据库Helper**
   - 使用 `projectManager.getDbHelperForProject(projectName)` 获取对应项目的数据库Helper
   - 验证Helper不为null

2. **加载Note基本信息**
   - 查询字段：`_id`, `content`, `timestamp`, `cost`, `is_pinned`
   - 查询条件：`_id = noteId`
   - 将查询结果转换为Note对象

3. **加载Note的所有Comment**
   - 使用 `dbHelper.getCommentsForNote(noteId)` 获取Cursor
   - 遍历Cursor，转换为Comment对象列表
   - 按时间正序排序（最旧的在前）

4. **加载Note的标签（如果有）**
   - 使用 `dbHelper.getTagsForNote(noteId)` 获取标签Cursor
   - 转换为Tag对象列表

5. **组装数据并显示**
   - 将Note、Comment列表、标签列表传递给对话框
   - 在UI线程中显示对话框

#### 性能考虑
- **后台线程加载**：所有数据库操作在后台线程执行
- **进度提示**：显示ProgressDialog，至少显示0.5秒
- **错误处理**：捕获所有异常，显示友好提示
- **资源释放**：及时关闭Cursor，释放数据库资源

## 四、分阶段实施计划

### 阶段一：双击检测机制实现（基础功能）
**目标：** 实现双击检测，区分单击和双击事件  
**预计时间：** 1小时  
**风险：** 低

#### 具体实施措施

**1.1 在MainActivity中添加双击检测相关变量**
- 位置：`MainActivity` 类的成员变量区域
- 变量：
  - `Handler clickHandler`: 用于延迟执行单击操作
  - `Runnable clickRunnable`: 单击延迟执行的Runnable
  - `long lastClickTime`: 上次点击时间
  - `int lastClickPosition`: 上次点击位置（用于判断是否为同一项）
  - `static final long DOUBLE_CLICK_DELAY = 300`: 双击间隔时间常量（毫秒）

**1.2 修改时间轴列表点击事件处理**
- 位置：`loadTimelineData()` 方法中，设置 `listView.setOnItemClickListener()` 的部分
- 修改内容：
  - 移除现有的简单点击处理逻辑
  - 添加双击检测逻辑
  - 跳过日期分割线项（`DATE_DIVIDER` 类型）
  - 记录点击时间和位置
  - 实现延迟执行机制

**1.3 实现双击检测逻辑**
- 判断逻辑：
  1. 检查是否为日期分割线，如果是则直接返回
  2. 获取当前点击时间和位置
  3. 如果距离上次点击时间小于 `DOUBLE_CLICK_DELAY` 且位置相同，认为是双击
  4. 如果是双击：取消单击延迟执行，执行双击操作，重置状态
  5. 如果是单击：记录点击时间和位置，延迟执行单击操作（如果在这期间没有双击）

**1.4 创建双击处理方法（占位）**
- 创建 `handleDoubleClick(Comment item)` 方法
- 暂时只打印日志或显示Toast，后续阶段实现具体逻辑

#### 验收标准
- 双击时间轴项可以触发双击事件
- 单击不会立即触发，延迟后执行（或取消）
- 日期分割线不响应双击
- 不同位置的点击不会误判为双击

#### 测试要点
- 测试双击响应速度（300ms内）
- 测试单击延迟取消（300ms后无第二次点击）
- 测试快速连续点击不同项的处理
- 测试日期分割线不响应

### 阶段三：数据加载方法实现（核心功能）
**目标：** 实现从数据库加载Note详情的功能  
**预计时间：** 2小时  
**风险：** 中

#### 具体实施措施

**3.1 创建加载Note基本信息的方法**
- 方法名：`loadNoteById(NoteDbHelper dbHelper, long noteId)`
- 位置：`MainActivity` 类中
- 实现逻辑：
  1. 使用 `dbHelper.getReadableDatabase()` 获取数据库
  2. 查询 `TABLE_NOTES` 表，字段包括：`_id`, `content`, `timestamp`, `cost`, `is_pinned`
  3. 查询条件：`_id = noteId`
  4. 将Cursor结果转换为Note对象
  5. 关闭Cursor
  6. 返回Note对象，如果不存在返回null

**3.2 创建加载Comment列表的方法**
- 方法名：`loadCommentsForNote(NoteDbHelper dbHelper, long noteId)`
- 位置：`MainActivity` 类中
- 实现逻辑：
  1. 使用 `dbHelper.getCommentsForNote(noteId)` 获取Cursor
  2. 遍历Cursor，转换为Comment对象列表
  3. 设置Comment的项目名称（从Note获取）
  4. 设置Comment的类型为 `TimelineItemType.COMMENT`
  5. 关闭Cursor
  6. 返回Comment列表

**3.3 创建加载标签的方法**
- 方法名：`loadTagsForNote(NoteDbHelper dbHelper, long noteId)`
- 位置：`MainActivity` 类中
- 实现逻辑：
  1. 使用 `dbHelper.getTagsForNote(noteId)` 获取标签Cursor
  2. 遍历Cursor，转换为Tag对象列表（或直接使用Cursor）
  3. 关闭Cursor
  4. 返回标签列表（或Cursor）

**3.4 创建主加载方法**
- 方法名：`loadNoteDetail(long noteId, String projectName)`
- 位置：`MainActivity` 类中
- 实现逻辑：
  1. 显示ProgressDialog（"正在加载详情..."）
  2. 在后台线程中执行：
     - 获取数据库Helper：`projectManager.getDbHelperForProject(projectName)`
     - 验证Helper不为null，否则抛出异常
     - 调用 `loadNoteById()` 加载Note基本信息
     - 验证Note不为null，否则抛出异常
     - 调用 `loadCommentsForNote()` 加载Comment列表
     - 调用 `loadTagsForNote()` 加载标签（如果有）
  3. 在UI线程中：
     - 关闭ProgressDialog（至少显示0.5秒）
     - 调用 `showNoteDetailDialog()` 显示详情（后续阶段实现）
  4. 错误处理：
     - 捕获所有异常
     - 在UI线程中显示Toast错误提示
     - 关闭ProgressDialog

**3.5 错误处理策略**
- Note不存在：返回null，显示"Note不存在"提示
- 项目不存在：返回null，显示"项目不存在"提示
- 数据库错误：捕获异常，显示"加载失败：错误信息"
- 空数据：Comment列表为空时正常显示，不报错

#### 验收标准

- 可以正确加载Note的所有Comment（按时间正序）
- 可以正确加载Note的标签（如果有）
- 错误情况有友好的提示
- 加载过程有进度提示（至少显示0.5秒）

## 五、技术实现细节

### 5.1 双击检测实现方案

#### 核心机制
使用Handler延迟检测机制：
- 第一次点击：记录点击时间和位置，延迟300ms执行单击操作
- 第二次点击（300ms内）：取消单击延迟执行，执行双击操作
- 延迟时间：300ms（可调整）

#### 实现要点
- 使用 `Handler` 和 `Runnable` 实现延迟
- 记录上次点击时间和位置，判断是否为同一项
- 跳过日期分割线项
- 重置状态机制

### 5.2 项目切换和详情加载流程

#### 流程设计
```
双击事件
  ↓
检查项目一致性
  ↓
[需要切换?]
  ├─ 是 → 切换项目 → 加载详情 → 显示对话框
  └─ 否 → 加载详情 → 显示对话框
  ↓
时间轴对话框保持打开（跨项目功能）
```

#### 关键方法调用链
1. `handleDoubleClick(Comment item)` - 处理双击事件
2. `switchProjectAndShowDetail(String projectName, long noteId)` - 切换项目并显示详情
3. `switchProject(String projectName)` - 切换项目（现有方法）
4. `loadNoteDetail(long noteId, String projectName)` - 加载Note详情
5. `showNoteDetailDialog(Note note, List<Comment> comments, Cursor tagsCursor)` - 显示详情对话框

### 5.3 Note详情数据加载方案

#### 数据加载流程
1. **获取数据库Helper**
   - 使用 `projectManager.getDbHelperForProject(projectName)`
   - 验证Helper不为null

2. **加载Note基本信息**
   - 查询 `TABLE_NOTES` 表
   - 字段：`_id`, `content`, `timestamp`, `cost`, `is_pinned`
   - 条件：`_id = noteId`
   - 转换为Note对象

3. **加载Comment列表**
   - 使用 `dbHelper.getCommentsForNote(noteId)` 获取Cursor
   - 遍历Cursor，转换为Comment对象列表
   - 设置Comment的项目名称和类型

4. **加载标签**
   - 使用 `dbHelper.getTagsForNote(noteId)` 获取Cursor
   - 转换为标签数据（可直接使用Cursor或转换为对象列表）

5. **组装并显示**
   - 在UI线程中显示对话框
   - 传递Note、Comment列表、标签数据

### 5.4 对话框布局设计要点

#### 布局层次
- 根布局：`LinearLayout`（vertical）
- 标题栏：固定高度，包含标题和关闭按钮
- Note内容区域：`ScrollView`，支持长内容滚动
- Comment列表：`ListView`，支持滚动
- 底部操作栏：固定高度，包含操作按钮

#### 样式规范
- 对话框大小：宽度90%，高度80%
- 标题栏：使用主题色背景，白色文字
- 内容区域：白色背景，适当padding
- 按钮：使用主题色，圆角设计
- 字体：标题18sp，内容14sp，时间11sp

### 5.5 错误处理策略

#### 错误类型和处理方式
1. **项目不存在**
   - 检测：`getDbHelperForProject()` 返回null
   - 处理：显示Toast "项目不存在"

2. **Note不存在**
   - 检测：`loadNoteById()` 返回null
   - 处理：显示Toast "Note不存在"

3. **数据库错误**
   - 检测：捕获SQLException
   - 处理：显示Toast "加载失败：错误信息"

4. **切换项目失败**
   - 检测：`switchProject()` 返回false或抛出异常
   - 处理：显示Toast "切换项目失败"
