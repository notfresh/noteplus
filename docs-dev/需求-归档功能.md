# 需求 - 归档功能（含归档时间字段）

## 1. 背景与目标
当前一个项目包含大量笔记，用户需要“归档”不常用的笔记，使其不在项目主列表显示，同时在设置页有入口查看已归档内容，并支持一键还原。

## 2. 核心需求
1. **归档**：对单条笔记进行归档操作。
2. **隐藏**：归档笔记不在项目主列表显示。
3. **入口**：设置页提供“已归档”入口。
4. **列表**：已归档列表可查看所有归档笔记。
5. **还原**：归档笔记支持还原，回到原项目主列表。
6. **归档时间**：每条归档笔记记录归档时间（可显示与排序）。

## 3. 业务规则
- 归档是**状态变更**，不是删除，所有内容（评论、标签、时间范围）保留。
- 归档后**笔记及其评论**在主列表与时间线中均被过滤。
- 归档列表默认按归档时间倒序；可选按笔记时间排序。
- 归档后不参与置顶与时间线（默认方案），避免影响主流程。
- 多项目独立数据库：归档状态与归档时间在项目内独立保存。

## 4. 数据结构设计
### 4.1 新增字段
在笔记表新增字段：
- `is_archived`：归档状态（0=未归档，1=已归档）
- `archived_at`：归档时间（时间戳，未归档为 0 或 NULL）

### 4.2 数据迁移
- 旧数据迁移时默认 `is_archived=0`，`archived_at=0`。
- 升级逻辑需兼容历史版本数据库。

## 5. UI/交互方案
### 5.1 项目主列表
- 默认过滤掉 `is_archived=1` 的记录。
- 归档笔记的评论不参与任何列表或时间线展示。
- 列表空状态提示可加“检查是否已归档”文案（可选）。

### 5.2 长按菜单
- 在笔记长按菜单新增“归档”。
- 归档后提示“已归档，可在设置-已归档中查看”。

### 5.3 设置入口
- 设置页新增“已归档”入口，点击进入归档列表。
- 可按项目过滤（建议），或显示当前项目归档列表。

### 5.4 归档列表
- 展示归档笔记，显示项目名（如果跨项目）与归档时间。
- 长按菜单提供“还原”。
- 还原后从归档列表移除，回到项目主列表。

## 6. 影响范围
- 列表加载逻辑（过滤归档，评论随笔记过滤）
- 笔记长按菜单（新增归档操作）
- 设置页（新增入口，放在项目设置里）
- 数据库升级（新增字段）
- 导入/导出（包含归档数据：导出时保留归档相关字段，导入时识别归档内容）

## 7. 关键场景
1. **归档后主列表即时更新，且评论随笔记一起隐藏**。
2. **归档列表还原后，回到原项目显示**。
3. **归档笔记在跨项目时间线中不展示，评论同样不展示**。
4. **导入/导出保留归档状态与归档时间（建议支持）**。

## 8. 实现步骤（开发计划）
1. **数据库升级**
   - 增加 `is_archived` 与 `archived_at` 字段。
   - 数据迁移脚本：旧数据默认未归档。

2. **数据访问层**
   - 列表查询增加过滤条件：`is_archived=0`。
   - 新增归档与还原的更新方法（更新状态与时间字段）。

3. **主列表与长按菜单**
   - 长按菜单新增“归档”操作。
   - 归档后刷新列表与提示。

4. **设置页入口**（放在项目设置里）
   - 新增“已归档”入口。
   - 进入归档列表页面/对话框。

5. **归档列表实现**
   - 加载归档笔记（`is_archived=1`）。
   - 展示归档时间、项目名（如需要）。
   - 长按菜单提供“还原”。

6. **跨项目与时间线**
   - 统一在跨项目查询中排除归档笔记（可选策略）。

7. **导入/导出（可选）**
   - 导出时包含 `is_archived` 与 `archived_at`。
   - 导入时正确恢复归档状态。

8. **测试与回归**
   - 归档/还原流程。
   - 过滤与列表刷新。
   - 项目切换后归档状态一致。
   - 数据升级兼容。

## 9. 可选增强（暂时不做）
- 归档列表内支持搜索与筛选。
- “批量归档/还原”。
- 归档列表显示“归档时间+原始创建时间”。
