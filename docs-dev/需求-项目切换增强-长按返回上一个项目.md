# 项目切换增强 - 长按2秒返回上一个项目

## 一、需求分析

### 1.1 功能描述
- 在项目切换按钮上，长按2秒后自动切换到上一个项目
- 类似于浏览器的"后退"功能，方便快速在两个项目之间切换

### 1.2 使用场景
- 用户在项目A和项目B之间频繁切换
- 误切换到其他项目，想快速返回
- 临时查看其他项目后，想快速回到之前的工作项目

## 二、设计方案

### 2.1 核心思路（简化版）

**上一个项目记录**：
- 使用一个全局变量保存"上一个项目"
- 切换项目时：将当前项目保存到变量，然后切换到新项目
- 长按返回时：如果变量不为空，切换到变量中保存的项目

**交互设计**：
- 在项目切换按钮上添加长按监听
- 长按开始：显示提示（如 Toast 或进度指示）
- 长按持续2秒：触发返回上一个项目
- 长按取消：不执行任何操作

### 2.2 数据结构设计（简化版）

#### 2.2.1 在 ProjectContextManager 中添加上一个项目记录

```java
public class ProjectContextManager {
    // 新增：上一个项目名称（全局变量）
    private String previousProjectName = null;
    
    /**
     * 获取上一个项目名称
     * @return 上一个项目名称，如果没有则返回 null
     */
    public String getPreviousProject() {
        return previousProjectName;
    }
    
    /**
     * 切换到指定项目（修改原有方法）
     */
    public boolean switchToProject(String projectName) {
        if (projectName == null || projectName.isEmpty()) {
            return false;
        }
        
        // 新增：在切换前，将当前项目保存为"上一个项目"
        // 只有在真正切换项目时才记录（不是首次设置，且不是切换到同一个项目）
        if (currentProjectName != null 
                && !currentProjectName.equals(projectName)) {
            previousProjectName = currentProjectName;
        }
        
        // 原有逻辑：切换项目
        currentProjectName = projectName;
        preferences.edit().putString(KEY_CURRENT_PROJECT, projectName).apply();
        addProjectToList(projectName);
        
        return true;
    }
    
    /**
     * 清除上一个项目记录（可选，在应用启动时调用）
     */
    public void clearPreviousProject() {
        previousProjectName = null;
    }
}
```

**说明**：
- 只需要一个 `String` 变量，非常简单
- 不需要持久化存储（应用重启后清空，符合预期）
- 切换逻辑清晰：切换前保存当前项目，切换后更新当前项目

            setupLongPressListener(child);

private void setupLongPressListener(View view) {
    final long LONG_PRESS_DURATION = 2000; // 2秒
    final Handler handler = new Handler();
    final Runnable longPressRunnable = new Runnable() {
        @Override
        public void run() {
            // 长按2秒后执行
            switchToPreviousProject();
        }
    };
    
    view.setOnTouchListener(new View.OnTouchListener() {
        private long pressStartTime = 0;
        private boolean isLongPressTriggered = false;
        
        @Override
        public boolean onTouch(View v, MotionEvent event) {
            switch (event.getAction()) {
                case MotionEvent.ACTION_DOWN:
                    pressStartTime = System.currentTimeMillis();
                    isLongPressTriggered = false;
                    // 延迟2秒后执行长按操作
                    handler.postDelayed(longPressRunnable, LONG_PRESS_DURATION);
                    // 显示提示
                    showLongPressHint();
                    return false; // 返回 false，让点击事件正常处理
                    
                case MotionEvent.ACTION_UP:
                case MotionEvent.ACTION_CANCEL:
                    // 取消长按操作
                    handler.removeCallbacks(longPressRunnable);
                    hideLongPressHint();
                    // 如果已经触发了长按，不再执行点击事件
                    if (isLongPressTriggered) {
                        return true; // 消费事件，不触发点击
                    }
                    return false; // 不消费事件，允许点击
            }
            return false;
        }
    });
}

**更简单的方案**：使用 `View.OnLongClickListener`（但无法精确控制2秒）：

```java
// 方案2：使用自定义长按监听器（精确控制2秒）
public class LongPressHandler implements View.OnTouchListener {
    private static final long LONG_PRESS_DURATION = 2000; // 2秒
    private Handler handler = new Handler();
    private Runnable longPressRunnable;
    private OnLongPressListener listener;
    private long pressStartTime;
    
    public interface OnLongPressListener {
        void onLongPress();
        void onPressStart(); // 长按开始
        void onPressCancel(); // 长按取消
    }
    
    public LongPressHandler(OnLongPressListener listener) {
        this.listener = listener;
        this.longPressRunnable = () -> {
            if (listener != null) {
                listener.onLongPress();
            }
        };
    }
    
    @Override
    public boolean onTouch(View v, MotionEvent event) {
        switch (event.getAction()) {
            case MotionEvent.ACTION_DOWN:
                pressStartTime = System.currentTimeMillis();
                handler.postDelayed(longPressRunnable, LONG_PRESS_DURATION);
                if (listener != null) {
                    listener.onPressStart();
                }
                return false; // 不消费事件，允许点击
                
            case MotionEvent.ACTION_UP:
            case MotionEvent.ACTION_CANCEL:
                handler.removeCallbacks(longPressRunnable);
                if (listener != null) {
                    listener.onPressCancel();
                }
                return false; // 不消费事件，允许点击
        }
        return false;
    }
}
```

```java
private ProgressDialog longPressProgress;

private void showLongPressHint() {
    String previousProject = projectManager.getPreviousProject();
    if (previousProject == null) {
        Toast.makeText(this, "没有上一个项目", Toast.LENGTH_SHORT).show();
        return;
    }
    
    longPressProgress = new ProgressDialog(this);
    longPressProgress.setMessage("长按返回: " + previousProject);
    longPressProgress.setProgressStyle(ProgressDialog.STYLE_HORIZONTAL);
    longPressProgress.setMax(100);
    longPressProgress.setProgress(0);
    longPressProgress.setCancelable(false);
    longPressProgress.show();
    
    // 启动进度更新
    startLongPressProgress();
}

private void startLongPressProgress() {
    final long duration = 2000; // 2秒
    final long interval = 50; // 每50ms更新一次
    final Handler handler = new Handler();
    final long startTime = System.currentTimeMillis();
    
    Runnable updateProgress = new Runnable() {
        @Override
        public void run() {
            if (longPressProgress == null || !longPressProgress.isShowing()) {
                return;
            }
            
            long elapsed = System.currentTimeMillis() - startTime;
            int progress = (int) (elapsed * 100 / duration);
            
            if (progress >= 100) {
                longPressProgress.dismiss();
            } else {
                longPressProgress.setProgress(progress);
                handler.postDelayed(this, interval);
            }
        }
    };
    
    handler.post(updateProgress);
}

private void hideLongPressHint() {
    if (longPressProgress != null && longPressProgress.isShowing()) {
        longPressProgress.dismiss();
    }
}
```

#### 2.3.3 切换到上一个项目

```java
/**
 * 切换到上一个项目（长按触发）
 */
private void switchToPreviousProject() {
    String previousProject = projectManager.getPreviousProject();
    
    // 如果变量为空，不予理睬
    if (previousProject == null) {
        Toast.makeText(this, "没有上一个项目", Toast.LENGTH_SHORT).show();
        return;
    }
    
    // 检查上一个项目是否还存在
    List<String> projects = projectManager.getProjectList();
    if (!projects.contains(previousProject)) {
        Toast.makeText(this, "上一个项目 \"" + previousProject + "\" 已不存在", 
            Toast.LENGTH_SHORT).show();
        // 清除无效的上一个项目记录
        projectManager.clearPreviousProject();
        return;
    }
    
    // 切换到上一个项目
    // 注意：switchProject 内部会更新 previousProjectName
    switchProject(previousProject);
    
    // 显示提示
    Toast.makeText(this, "已返回: " + previousProject, Toast.LENGTH_SHORT).show();
}
```

### 2.4 边界情况处理

1. **没有上一个项目**（变量为空）：
   - 显示提示："没有上一个项目"
   - 不执行任何操作（不予理睬）

2. **上一个项目已被删除**：
   - 显示提示："上一个项目已不存在"
   - 清除上一个项目记录（设为 null）

3. **首次使用或应用重启**：
   - 变量初始化为 null
   - 需要至少切换过一次项目才有上一个项目记录
   - 应用重启后，上一个项目记录会清空（符合预期）

4. **长按过程中取消**：
   - 用户抬起手指或移动手指
   - 取消长按操作，不切换项目

5. **快速连续长按**：
   - 防止重复触发
   - 使用标志位控制

6. **切换到同一个项目**：
   - 不会更新上一个项目记录（避免覆盖）

## 六、注意事项

1. **变量初始化**：`previousProjectName` 初始化为 `null`，应用重启后自动清空
2. **事件处理**：长按监听不能阻止正常的点击事件（返回 `false`）
3. **切换逻辑**：只有在真正切换项目时才更新 `previousProjectName`（不是首次设置，且不是切换到同一个项目）
4. **数据一致性**：项目删除时，如果删除的是 `previousProjectName`，需要清除该变量
5. **用户体验**：长按时间1秒要合适，太短容易误触，太长体验不好
6. **View 查找**：需要根据实际布局结构调整查找项目切换按钮的方式（可能需要通过 Toolbar 遍历子 View 或使用其他标识）
