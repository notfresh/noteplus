# 评论导入导出方案

## 一、JSON格式

### 导出格式
```json
{
  "notes": [
    {
      "id": 1,
      "content": "笔记内容",
      "comments": [
        {
          "commentId": 1,
          "parentCommentId": null,
          "content": "评论内容",
          "timestamp": "2025-01-10 16:00:00",
          "cost": 0.0
        }
      ]
    }
  ]
}
```

### 实现要点
- **导出**：在 `writeJsonData()` 中，为每个笔记查询评论并添加到 `comments` 数组
- **导入**：在 `readJsonData()` 中，解析 `comments` 数组，维护ID映射，按时间顺序导入

---

## 二、CSV格式（兼容旧格式）

### 新格式（带评论）
```
类型,笔记ID,评论ID,父评论ID,内容,时间戳,花费,开始时间,结束时间,标签
NOTE,1,,"笔记内容","2025-01-10 15:20:00",10.5,"2025-01-10 10:00:00","2025-01-10 12:00:00","tag1"
COMMENT,1,1,,"评论1","2025-01-10 16:00:00",0.0,,,
COMMENT,1,2,1,"回复评论1","2025-01-10 16:30:00",5.0,,,
```

### 旧格式（无评论）
```
ID,内容,时间戳,花费,开始时间,结束时间,标签
1,"笔记内容","2025-01-10 15:20:00",10.5,"2025-01-10 10:00:00","2025-01-10 12:00:00","tag1"
```

### 兼容判断
- **导出**：检查笔记是否有评论，有则用新格式，无则用旧格式
- **导入**：读取第一行头部，判断是否有"类型"列
  - 有"类型"列 → 新格式，按类型分别处理NOTE和COMMENT
  - 无"类型"列 → 旧格式，只导入笔记

### 实现要点
- **导出**：在 `writeCsvData()` 中，先写笔记行（类型=NOTE），再写评论行（类型=COMMENT）
- **导入**：在 `readCsvData()` 中，根据头部判断格式，两遍扫描（先NOTE后COMMENT）

---

## 三、实现步骤

1. **JSON导出**：修改 `writeJsonData()`，添加评论查询和序列化
2. **JSON导入**：修改 `readJsonData()`，解析评论并维护ID映射
3. **CSV导出**：修改 `writeCsvData()`，支持新格式（带类型列）
4. **CSV导入**：修改 `readCsvData()`，根据头部判断格式并兼容处理
