# 全文检索功能设计文档

## 1. 概述

为 NotePlus 应用添加基于 Lucene + IK Analyzer 的中文全文检索功能，实现笔记正文的实时分词搜索。

## 2. 技术选型

| 组件 | 选型 | 版本 |
|------|------|------|
| 搜索引擎核心 | Lucene Core | 9.x |
| 中文分词器 | IK Analyzer | 2024.0 |
| 最低 Android SDK | 29 (Android 10) | - |

**包体积增加**: 约 3-4MB

## 3. 架构设计

### 3.1 核心组件

┌─────────────────┐ ┌──────────────┐ ┌──────────────┐

## 4. 数据流设计

### 4.1 索引构建时机

| 场景 | 触发时机 |
|------|----------|
| 新建笔记 | 笔记保存成功后立即索引 |
| 编辑笔记 | 笔记保存成功后更新索引 |
| 删除笔记 | 删除成功后从索引中移除 |
| 应用首次安装 | 后台异步构建全部索引 |
| 索引损坏/丢失 | 自动检测并重建 |

### 4.2 存量笔记索引检测

为确保存量笔记全部可被检索，系统启动时进行一致性检测：

```
应用启动
    ↓
检查索引目录是否存在
    ↓
┌─────────────────────────────────────────┐
│ 索引不存在 / 损坏 / 版本不匹配           │
│     ↓                                   │
│ 触发全量索引构建                         │
│ （后台 WorkManager 执行）                │
└─────────────────────────────────────────┘
┌─────────────────────────────────────────┐
│ 索引存在且正常                           │
│     ↓                                   │
│ 查询 notes 表 LEFT JOIN                 │
│ search_index_status 表                  │
│     ↓                                   │
│ WHERE search_index_status.note_id IS NULL
│ → 未索引笔记，批量构建                   │
│     ↓                                   │
│ 版本不一致 → 增量更新缺失部分             │
│ 版本一致 → 直接使用                       │
└─────────────────────────────────────────┘
```

**版本号记录**: 在 `SharedPreferences` 中记录 `search_index_version`，与数据库版本对齐。每次数据库版本升级时递增，触发索引重建。

### 4.3 搜索流程

```
用户输入搜索词
    ↓
SearchService.tokenize() → IK 分词处理 → ["关键词1", "关键词2", ...]
    ↓
构建 BooleanQuery（各分词 OR 匹配）
    ↓
执行搜索，返回匹配文档及分词位置
    ↓
SearchResult 高亮处理
    ↓
展示结果列表
```

## 5. 索引结构设计

### 5.1 索引记录表

```sql
CREATE TABLE search_index_status (
    note_id INTEGER PRIMARY KEY,
    indexed_at INTEGER NOT NULL,
    FOREIGN KEY (note_id) REFERENCES notes(_id) ON DELETE CASCADE
);
```

| 字段名 | 类型 | 说明 |
|--------|------|------|
| note_id | INTEGER | 笔记 ID（关联 notes 表） |
| indexed_at | INTEGER | 索引构建时间戳 |

**作用**：精确记录每条笔记是否已建索引，启动时与 notes 表做 LEFT JOIN 比对，快速定位未索引的笔记。

### 5.2 Lucene 索引字段

### 5.2 分词配置

- **IK 模式**: `IK_MAX_WORD`（细粒度分词，最大覆盖）
- **搜索时**: `IK_SMART`（智能分词，提高精度）

## 7. 索引管理

### 7.1 索引存储位置

```
/data/data/person.notfresh.noteplusv2/files/search_index/
```

### 7.2 索引一致性保证

| 机制 | 说明 |
|------|------|
| 索引记录表 | `search_index_status` 表记录每条笔记的索引状态 |
| 版本号校验 | SharedPreferences 存储 `search_index_version`，与 DATABASE_VERSION 对齐 |
| 启动检测 | 应用启动时比对 notes 与 search_index_status，快速定位未索引笔记 |
| 增量同步 | 笔记 CRUD 操作实时更新索引，版本不一致时补全缺失部分 |

### 7.3 后台索引构建

- 使用 `WorkManager` 进行后台索引构建
- 构建过程中显示进度（可选）
- 不阻塞主线程和日常使用

## 8. 错误处理

| 场景 | 处理方式 |
|------|----------|
| 索引写入失败 | 记录日志，下次保存时重试 |
| 搜索失败 | Toast 提示用户，显示错误信息 |
| 索引文件损坏 | 自动删除并重建 |
| 无匹配结果 | 显示"未找到相关笔记" |

## 9. 性能考虑

- **索引延迟写入**: 笔记保存后异步写入索引，不阻塞 UI
- **搜索防抖**: 输入完成后 300ms 再执行搜索，避免频繁查询
- **结果上限**: 单次搜索最多返回 100 条结果
